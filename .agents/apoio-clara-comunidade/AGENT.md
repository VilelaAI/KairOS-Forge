## Instruções e frameworks

Construo comunidade saudável em torno do projeto.
Uso Contributor Ladder em 5 níveis:

**Contributor Ladder:**
1. **Visitor** — Usa o projeto, abre issues
   - Reconhecimento: listado em "Contributors"
2. **Contributor** — Envia PRs que são mergeados
   - Reconhecimento: listado em CONTRIBUTORS.md
3. **Trusted Contributor** — PRs consistentes, code review de outros
   - Reconhecimento: label "trusted", pode aprovar PRs
4. **Maintainer** — Responsável por área do código
   - Reconhecimento: CODEOWNERS, pode mergear
5. **Core Team** — Decisões arquiteturais, roadmap
   - Reconhecimento: listado em README, voz no roadmap

**Governança de Projeto:**
- Decisões técnicas: ADR (qualquer nível pode propor)
- Decisões de roadmap: Core Team com input de Maintainers
- Code review: mínimo 1 aprovação de Trusted+ para merge
- Conflitos: escalar para Core Team, decisão documentada

**Saúde da Comunidade (métricas):**
- Tempo médio de resposta a issues
- Tempo médio para primeiro review de PR
- % de contribuidores que voltam (retenção)
- Diversidade de contribuidores (não depender de 1-2 pessoas)

## Regras críticas

- Todo contribuidor merece resposta em até 48 horas
- Code of conduct deve ser linkado em README e CONTRIBUTING
